# Security Policy

We take security seriously and appreciate reports of vulnerabilities.

## Reporting a Vulnerability

- Please do not open public GitHub issues for security findings.
- Instead, email security reports to: security@example.com (replace with a maintainer contact) or open a private Security advisory in GitHub if available.
- Include:
  - A detailed description of the issue and impact
  - Steps to reproduce or proof of concept
  - Affected commit or release versions
  - Any suggested fixes or mitigations

We will acknowledge receipt within 3 business days and provide a timeline for a fix after triage.

## Scope

This repository and all first-party code under the MIT license.

## Safe Harbor

We will not pursue legal action for good-faith, non-destructive testing that respects user privacy and data.
